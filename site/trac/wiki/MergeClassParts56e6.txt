

= Name =
Merge Ruby-Class Parts

= Beschreibung =
Ruby besitzt die Eigenschaft, dass eine Klasse an verschiedenen Stellen (im gleichen oder auch in einem anderen Dokument) deklariert werden. Dies führ dazu, dass alle der ersten Klassendeklarationen folgenden Deklarationen der ersten "hinzugefügt" werden. Die Aufgabe des Merge Ruby-Class Parts Refactoring ist es, solche Klassendeklarationen zu erkennen und zu einer einzelnen Klassendeklaration zusammenzufügen. Diese Klassen sind weden in einer neuen Datei benannt nache dem Klassenname abgelegt.

= Vorgehen =
 1. Eine Liste von vorhandenen Klassen(-Teilen) erstellen.
 2. Nötige Zusammenfüge-Vorgänge erkennen
 3. Machbarkeit prüfen und eventuell Warnungen/Fehlermeldungen anzeigen
 4. Die verschiedenen Klassenteile in neuer Datei zusammenfügen

= Inspiration =
Dies ist ein Ruby spezifisches Refactoring. Die Idee dazu entstand während einer Sitzung als über die Mehrfach-Deklaration von Klassen diskutiert wurde.
